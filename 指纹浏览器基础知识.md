# 指纹浏览器基础知识详解

---

## 一、什么是指纹浏览器

**指纹浏览器**是一种可以修改或伪装浏览器指纹信息的工具。通过伪造浏览器的各项底层特征（Canvas、WebGL、UserAgent、字体、屏幕分辨率等），使每个浏览器实例看起来像一台完全不同的真实设备。

### 1.1 浏览器指纹 vs 指纹浏览器

| 概念 | 说明 |
|------|------|
| **浏览器指纹** | 网站收集的浏览器软硬件特征组合，用于唯一标识设备 |
| **指纹浏览器** | 主动修改这些特征的软件，用于伪装身份、保护隐私 |

### 1.2 核心原理

```
真实设备
  └── Chrome on macOS → 固定指纹 (A)
  
指纹浏览器
  ├── 实例1 → 伪装 Win10 + Chrome → 指纹 (B)
  ├── 实例2 → 伪装 macOS + Safari → 指纹 (C)
  └── 实例3 → 伪装 Android + Chrome → 指纹 (D)
```

## 二、浏览器指纹的采集维度

指纹浏览器要伪造的不是一两个值，而是数十个维度的完整环境。

### 2.1 JavaScript 可读取的维度

| 维度 | 采集方式 | 示例值 |
|------|----------|--------|
| **User-Agent** | `navigator.userAgent` | `Mozilla/5.0 (Windows NT 10.0; Win64; x64)...` |
| **Platform** | `navigator.platform` | `Win32` / `MacIntel` / `Linux x86_64` |
| **语言** | `navigator.language` | `zh-CN` |
| **语言列表** | `navigator.languages` | `['zh-CN', 'zh', 'en']` |
| **时区** | `Intl.DateTimeFormat().resolvedOptions().timeZone` | `Asia/Shanghai` |
| **屏幕分辨率** | `screen.width` × `screen.height` | `1920×1080` |
| **可用分辨率** | `screen.availWidth` × `screen.availHeight` | `1920×1040` |
| **色深** | `screen.colorDepth` | `24` |
| **像素比** | `window.devicePixelRatio` | `2` (Retina) / `1` (普通) |
| **CPU核心数** | `navigator.hardwareConcurrency` | `8` |
| **内存大小** | `navigator.deviceMemory` | `8` (GB) |
| **触摸支持** | `navigator.maxTouchPoints` | `0` (桌面) / `5` (移动) |
| **Do Not Track** | `navigator.doNotTrack` | `1` / `null` |
| **插件列表** | `navigator.plugins` | Chrome PDF Viewer, Chrome PDF Plugin, Native Client |
| **MIME类型** | `navigator.mimeTypes` | 数十种 MIME 类型列表 |
| **Cookie 启用** | `navigator.cookieEnabled` | `true` |
| **线上/离线** | `navigator.onLine` | `true` |
| **Webdriver** | `navigator.webdriver` | `true`(自动化) / `false`(正常) |

### 2.2 Canvas 指纹

```javascript
// 网站采集 Canvas 指纹的原理
function getCanvasFingerprint() {
    const canvas = document.createElement('canvas');
    canvas.width = 200;
    canvas.height = 50;
    const ctx = canvas.getContext('2d');
    
    // 渲染特定文字
    ctx.textBaseline = 'top';
    ctx.font = '14px Arial';
    ctx.fillStyle = '#f60';
    ctx.fillRect(125, 1, 62, 20);
    ctx.fillStyle = '#069';
    ctx.fillText('Browser Fingerprint', 2, 15);
    
    // 取 PNG Data URL
    const dataUrl = canvas.toDataURL();
    // 转为哈希 → 不同设备/显卡/驱动 → 不同哈希
    return hash(dataUrl);
}
```

**指纹差异来源**：
- 操作系统字体渲染引擎（macOS vs Windows 渲染差异大）
- GPU 驱动差异（抗锯齿算法不同）
- 像素级别的颜色处理差异

### 2.3 WebGL 指纹

```javascript
function getWebGLFingerprint() {
    const canvas = document.createElement('canvas');
    const gl = canvas.getContext('webgl');
    
    // 获取 GPU 信息
    const debugInfo = gl.getExtension('WEBGL_debug_renderer_info');
    const vendor = gl.getParameter(debugInfo.UNMASKED_VENDOR_WEBGL);   // 厂商
    const renderer = gl.getParameter(debugInfo.UNMASKED_RENDERER_WEBGL); // 显卡型号
    
    // 示例：
    // vendor:  "Google Inc. (Apple)"
    // renderer: "ANGLE (Apple, ANGLE Metal Renderer: Apple M1, Unspecified Version)"
    
    return { vendor, renderer, /* GPU 独一无二 */ };
}
```

### 2.4 AudioContext 指纹

```javascript
function getAudioFingerprint() {
    const ctx = new (window.AudioContext || window.webkitAudioContext)();
    const oscillator = ctx.createOscillator();
    const analyser = ctx.createAnalyser();
    const gain = ctx.createGain();
    
    oscillator.type = 'triangle';
    oscillator.frequency.value = 10000;
    gain.gain.value = 0; // 静音
    
    oscillator.connect(analyser);
    analyser.connect(gain);
    gain.connect(ctx.destination);
    
    oscillator.start(0);
    
    // 读取音频数据（不同硬件有微小差异）
    const data = new Float32Array(analyser.fftSize);
    analyser.getFloatTimeDomainData(data);
    
    oscillator.stop();
    return hash(Array.from(data));
    // 不同声卡/DAC 产生不同结果
}
```

## 三、指纹浏览器如何伪造

### 3.1 架构：Chromium 源码修改 vs 浏览器扩展

```
方式一：Chromium 源码修改（主流商业方案）
  ├── 修改 Blink 渲染引擎源码
  ├── 直接替换 Navigator 对象的返回值
  ├── 重写 Canvas/WebGL/Audio 底层实现
  └── 优势：检测不到，因为是浏览器本身的返回值

方式二：JavaScript 注入（开源方案）
  ├── 页面加载前注入 JS
  ├── 重写 navigator/screen 等属性的 getter
  └── 劣势：可能被检测到（属性 descriptor 特征异常）
```

### 3.2 各维度的伪造方式

| 维度 | 伪造方式 | 关键技术 |
|------|----------|----------|
| **UserAgent** | 修改 `navigator.userAgent` | 需同时修改 HTTP Header |
| **Platform** | 修改 `navigator.platform` | 与 UA 逻辑一致 |
| **Canvas** | 注入随机噪声像素 | 每次随机 flip 最低位，哈希变化但不影响视觉效果 |
| **WebGL** | 替换 vendor/renderer 字符串 | 从真实设备池中选取匹配的组合 |
| **AudioContext** | 添加随机偏移 | 在音频数据中注入 ±0.00001 级的噪声 |
| **字体** | 随机增减字体列表 | 需要模拟不同系统的典型字体集合 |
| **屏幕分辨率** | 修改 `screen.width/height` | 同时修改 `availWidth/availHeight` |
| **时区** | 修改 `timeZone` 返回值 | 需与 IP 地理位置的时区匹配 |
| **WebRTC** | 设置 `iceTransportPolicy` 为 `relay` | 或直接禁用 WebRTC |
| **浏览器插件** | 修改 `navigator.plugins` | 数组需要可迭代，有 `namedItem` 方法 |

### 3.3 Canvas 噪声注入示例

```javascript
// 指纹浏览器注入的 Canvas 噪声逻辑
(function() {
    const origToDataURL = HTMLCanvasElement.prototype.toDataURL;
    const origToBlob = HTMLCanvasElement.prototype.toBlob;
    const origGetImageData = CanvasRenderingContext2D.prototype.getImageData;
    
    // 仅在 toDataURL/toBlob 时注入噪声
    // getImageData 保持原样（避免破坏页面功能）
    
    function addNoise(canvas) {
        const ctx = canvas.getContext('2d', { willReadFrequently: true });
        if (!ctx || canvas.width === 0) return;
        
        try {
            const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
            for (let i = 0; i < imageData.data.length; i += 4) {
                // 随机翻转 RGB 的最低位（肉眼不可见）
                imageData.data[i] ^= (Math.random() < 0.5 ? 1 : 0);
                imageData.data[i + 1] ^= (Math.random() < 0.5 ? 1 : 0);
            }
            ctx.putImageData(imageData, 0, 0);
        } catch(e) {}
    }
    
    HTMLCanvasElement.prototype.toDataURL = function(...args) {
        addNoise(this);
        return origToDataURL.apply(this, args);
    };
    
    HTMLCanvasElement.prototype.toBlob = function(callback, ...args) {
        addNoise(this);
        return origToBlob.call(this, callback, ...args);
    };
})();
```

```javascript
            devicePixelRatio: 1,
        },
        {
            vendor: 'Google Inc. (NVIDIA)',
            renderer: 'ANGLE (NVIDIA, NVIDIA GeForce RTX 3060 Direct3D11 vs_5_0 ps_5_0)',
            platform: 'Win32',
            devicePixelRatio: 1,
        },
    ];
    
    const profile = deviceProfiles[Math.floor(Math.random() * deviceProfiles.length)];
    
    WebGLRenderingContext.prototype.getParameter = function(parameter) {
        // UNMASKED_VENDOR_WEBGL = 37445
        if (parameter === 37445) return profile.vendor;
        // UNMASKED_RENDERER_WEBGL = 37446
        if (parameter === 37446) return profile.renderer;
        return getParameter.call(this, parameter);

    // WebGL2 也需处理
    if (typeof WebGL2RenderingContext !== 'undefined') {
        const getParameter2 = WebGL2RenderingContext.prototype.getParameter;
        WebGL2RenderingContext.prototype.getParameter = function(parameter) {
            if (parameter === 37445) return profile.vendor;
            if (parameter === 37446) return profile.renderer;
            return getParameter2.call(this, parameter);
        };
    }
})();
```

## 四、环境一致性——指纹浏览器的核心难点

### 4.1 什么是环境一致性

伪造一个维度容易，**让所有维度逻辑自洽**才是关键。

```
❌ 错误示例（不一致→被识别为指纹浏览器）
  UA: Chrome on Windows
  Platform: MacIntel
  WebGL Renderer: Apple M1 GPU
  时区: Asia/Tokyo
  语言: en-US
  → 多个维度互相矛盾

✅ 正确示例（自洽→像真实设备）
  UA: Chrome 120 on macOS
  Platform: MacIntel
  WebGL Renderer: Apple M1 GPU
  时区: Asia/Shanghai
  语言: zh-CN, zh
  像素比: 2
  分辨率: 1680×1050
  → 完美的 MacBook Pro 特征
```

### 4.2 需要校验的一致性规则

```python

# 1680×1050 + DPR 2 → MacBook Pro
# 2560×1440 + DPR 2 → MacBook Pro Retina
```

### 6.2 User Data Dir 隔离

```python
# 每个实例使用独立的数据目录，Cookie/Storage 完全隔离
import shutil
import tempfile
import os

class IsolatedBrowser:
    """隔离的浏览器实例"""
    
    def __init__(self, instance_id):
        self.instance_id = instance_id
        self.data_dir = os.path.join(
            tempfile.gettempdir(),
            f'chrome_instance_{instance_id}'
        )
        os.makedirs(self.data_dir, exist_ok=True)
    
    def launch(self):
        args = [
            f'--user-data-dir={self.data_dir}',
            '--no-first-run',
            '--no-default-browser-check',
            '--disable-background-networking',
            '--disable-sync',
            '--disable-default-apps',
        ]
        
        with sync_playwright() as p:
            context = p.chromium.launch_persistent_context(
                user_data_dir=self.data_dir,
                headless=True,
                args=args,
                # 指纹配置...
            )
            return context
    
    def cleanup(self):
        if os.path.exists(self.data_dir):
            shutil.rmtree(self.data_dir)

# 使用
instance1 = IsolatedBrowser('shop_001')
instance2 = IsolatedBrowser('shop_002')
# 两个实例的 Cookie、缓存完全独立
```

### 7.2 代理质量检测

```python
import requests
import json

def check_proxy_quality(proxy):
    """检测代理质量"""
    results = {}
    
    # 1. 连通性检测
    try:
        resp = requests.get('https://httpbin.org/ip', proxies=proxy, timeout=10)
        results['ip'] = resp.json()['origin']
        results['reachable'] = True
    except:
        return {'status': 'dead'}
    
    # 2. 匿名性检测
    headers = requests.get(
        'https://httpbin.org/headers', proxies=proxy, timeout=10
    ).json()['headers']
    
    # 透明代理会暴露 X-Forwarded-For
    results['anonymous'] = 'X-Forwarded-For' not in headers
    results['via_header'] = headers.get('Via', None)
    
    # 3. 地理位置
    geo = requests.get(
        f'http://ip-api.com/json/{results["ip"]}', timeout=10
    ).json()
    results['country'] = geo.get('country', 'Unknown')
    results['city'] = geo.get('city', 'Unknown')
    results['timezone'] = geo.get('timezone', 'Unknown')
    
    # 4. 速度测试
    import time
    start = time.time()
    requests.get('https://httpbin.org/get', proxies=proxy, timeout=10)
    results['latency_ms'] = int((time.time() - start) * 1000)
    
    return results
```

```html
    } catch(e) { report.webglError = e.message; }

        analyser.connect(gain);
        gain.connect(audioCtx.destination);
        osc.start(0);

async function digestMessage(message) {
    const msgUint8 = new TextEncoder().encode(message);
    const hashBuffer = await crypto.subtle.digest('SHA-256', msgUint8);
    return Array.from(new Uint8Array(hashBuffer))
        .map(b => b.toString(16).padStart(2, '0')).join('');
}
```

## 十、总结

| 层级 | 内容 | 难度 |
|------|------|------|
| 入门 | 修改 UA + 分辨率 + 时区 | ★☆☆☆☆ |
| 基础 | + Canvas 噪声 + WebGL 伪装 | ★★☆☆☆ |
| 进阶 | + AudioContext + 字体 + WebRTC | ★★★☆☆ |
| 核心 | **环境一致性**（所有维度逻辑自洽） | ★★★★☆ |
| 高阶 | + 行为模拟 + IP 绑定 + 独立存储 | ★★★★★ |

**核心原则**：指纹浏览器的目标不是"把所有值改成随机"，而是"伪装成一个逻辑自洽的真实设备"。一致性是灵魂，随机性是敌人。
